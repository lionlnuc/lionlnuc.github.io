配置环境、创建项目与导航栏

一、目标

完成项目从 0 到 1 的搭建：

  • 配置 Python、Node.js、Git、IDE 等开发环境；
  • 创建 Django 后端项目与 Vue 3 前端项目；
  • 配置 DRF、JWT、CORS、静态文件和媒体文件；
  • 使用 Tailwind CSS + DaisyUI 搭建基础导航栏；
  • 理解开发环境与生产部署环境的区别；
  • 完成前端打包到 Django 后端的基础整合。

本节是整个 AgentTalk 项目的地基，后续登录、角色、聊天、语音模块都建立在这里的前后端工程结构之上。

后端使用 Django，因此需要安装 Python，并建议使用虚拟环境隔离依赖。

十、前后端整合流程

10.1 开发环境

开发时通常启动两个服务：

访问：

  • 前端：http://localhost:5173
  • 后端：http://127.0.0.1:8000

此时前后端通过 CORS 通信。

10.2 生产 / 打包环境

构建后：

  1. Vue 源码被编译成 JS、CSS、图片等静态资源；
  2. 静态资源输出到 Django 的 static/frontend；
  3. Django 返回 index.html；
  4. 浏览器加载静态资源并启动 Vue 应用。

10.3 完整访问生命周期

  1. 用户访问 http://127.0.0.1:8000/；
  2. Django 路由匹配根路径，返回 index.html；
  3. 浏览器解析 HTML，请求 JS / CSS 静态资源；
  4. Vue 应用启动，接管页面渲染；
  5. 用户操作页面时，Vue 调用 Django API 获取数据；
  6. Django 查询数据库或调用大模型服务，返回 JSON / SSE 数据。

十一、Git 忽略文件建议

  • 本地数据库；

  • node_modules；
  • 大体积模型文件。

十三、小结

本节完成了项目基础骨架：

  • Django 提供后端 API、数据库、静态资源服务；
  • Vue 负责 SPA 页面和用户交互；
  • Vite 负责开发热更新与生产打包；
  • Tailwind CSS + DaisyUI 提供 UI 能力；
  • Vue Router 负责前端页面切换；
  • Django 的兜底路由解决 History 模式刷新问题。

后续所有模块都围绕这套结构继续扩展：登录模块负责用户身份，角色模块负责内容生产，信息流负责展示，聊天模块负责 AI 交互，语音模块负责自然交互体验。

二十一、小练习与自测答案

练习 1

问题：为什么刷新 /friend/ 页面时不能直接让 Django 查找一个后端页面？

答案：/friend/ 是 Vue Router 的前端路由，不是 Django 业务接口。刷新时 Django 需要返回同一个 index.html，再由 Vue 根据 URL 渲染好友页面。

练习 2

问题：为什么兜底路由要排除 /media/ 和 /assets/？

答案：这些路径是真实资源路径。如果不排除，浏览器请求图片或 JS 文件时会拿到 index.html，导致图片无法显示或页面白屏。

练习 3

问题：为什么导航栏适合做成公共组件？

答案：导航栏在多个页面都会出现。做成组件后，只维护一份代码，所有页面都能复用。
